库存写入

1 填端点 请求地址填供应商给的 Base URL；根地址会自动尝试补 `/v1`。
2 粘 Key 一行一个 Key，或粘 JSON 数组；服务端会逐条校验格式和连通性。
3 一键获取模型 模型留空时自动请求 `/models`，不支持时再用最小请求探测。
4 写入可用库存 只写入探测成功的 Key；失败原因会显示在本页，不回传明文 Key。
5 自动切换 单个渠道认证失败、额度耗尽或 5xx 后，网关自动切到同模型可用渠道。
