# 记忆索引总目录 (Master Index)

> 最后更新: 2026-03-15
> 用途: 所有记忆/日志的一级分类入口。处理任何问题前，先读此文件确定相关领域。

## 调度流程

```
1. 读 INDEX.md (本文件) → 确定一级分类
2. 进入对应目录 → 读该目录的 INDEX.md → 确定具体编号
3. 读编号对应的文件 → 获取详细信息
4. 复杂问题 → 并行查多个分类索引 (多调度)
```

## 分类总览

| 编号 | 分类 | 前缀 | 路径 | 说明 |
|------|------|------|------|------|
| 01 | 社交媒体 | SOC | `memory/social/` | 发布记录、内容创作、互动策略、人设 |
| 02 | 系统运维 | SYS | `memory/system/` | 网关配置、服务状态、基础设施变更 |
| 03 | 交易盈利 | TRD | `memory/trading/` | 交易记录、PnL、风控、策略迭代 |
| 04 | 开发任务 | DEV | `memory/development/` | 功能开发、代码变更、技术决策 |
| 05 | 日常运营 | OPS | `memory/operations/` | 心跳检查、cron任务、检索日志 |
| 06 | 错误处理 | ERR | `memory/errors/` | 失败记录、异常处理、修复方案 |
| 07 | 每日日志 | DAY | `memory/daily/` | 原始日记 YYYY-MM-DD.md (不编号) |

## 编号规则

- 格式: `{前缀}-{三位序号}` (如 `SOC-001`, `ERR-012`)
- 每个分类独立编号，从 001 开始
- 文件名 = 编号 + 简短描述: `SOC-001-publish-runs.md`
- 每个文件内部条目用 `## {编号}.{子序号}` 标记 (如 `## SOC-001.1`)

## 交叉引用

条目间可用 `→ 参见 {编号}` 互相引用。例如:
- SOC-001 中的发布失败 → 参见 ERR-001
- DEV-002 中的功能改动 → 参见 SYS-001

## 多调度示例

处理"社交发布失败"问题时，应同时调度:
1. `SOC-001` (发布记录) — 了解发布历史
2. `ERR-001` (失败记录) — 了解错误详情
3. `SYS-001` (系统配置) — 确认环境状态
