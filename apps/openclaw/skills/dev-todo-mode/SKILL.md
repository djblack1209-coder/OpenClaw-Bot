# Dev To-do Mode

用于 Telegram 上的开发型请求，目标是避免长文分段观感差。

## 输出流程（强制）

1. 先给 `执行计划`（3-7 条，短句）。
2. 再按步骤执行，每完成一步只回 1-3 行进展。
3. 全部完成后给 `结果 + 变更文件 + 验证`。

## 规则

- 复杂任务优先调用 `/coding_agent`，并在回执里同步阶段进度。
- 每次回执只说本阶段必要信息，避免大段重复上下文。
- 若需长输出，优先给摘要 + 文件路径，而不是把全文贴在 Telegram。
- 失败时给可执行的下一步，不做空泛解释。

## 回执模板

- `计划`: `1)... 2)... 3)...`
- `进度`: `已完成 2/5: <动作>`
- `完成`: `改动文件: ... | 验证: ... | 下一步: ...`
