# OpenClaw APP 全量视觉级审计提示词

> 用法：新建会话，把下面整段提示词原样发给 AI。目标是做“逐页、逐组件、逐交互”的视觉级审计，而不是只跑接口和编译。

---

你现在要对 **OpenClaw APP** 做一次 **全量视觉级审计**。

## 审计目标

这次不是做接口冒烟，不是只看 curl 返回 200，也不是只跑编译。

你必须：

1. **逐个页面** 检查
2. **逐个组件** 检查
3. **逐个关键交互** 检查
4. 给出 **真实页面结果**，不是代码推测
5. 对每个问题给出：
   - 页面
   - 组件
   - 现象
   - 预期
   - 严重度
   - 是否可复现

## 硬性要求

### 1. 必须先做环境基线

先验证：

```bash
curl -s http://127.0.0.1:18790/api/v1/status | python3 -m json.tool
curl -s http://127.0.0.1:18790/api/v1/system/services | python3 -m json.tool
curl -s http://127.0.0.1:1420
```

如果后端或前端没起来，先明确说清楚，不要假装审计通过。

### 2. 必须用真实页面交互

你要用真实浏览器或 Playwright，**点击页面本身**，不能只读代码然后说“应该能跳”。

### 3. 不准只用接口代替页面审计

例如：
- AI 助手消息发送，必须真实输入并点击发送
- 设置页语言切换，必须真实切换中英文
- 插件商店审批/拒绝，必须真实点击按钮
- 世界地图 hover，必须真实 hover 验证

### 4. 不准说“应该”“看起来”“大概率”

只能写：
- ✅ 已验证
- ❌ 已复现
- ⏭️ 无法验证（说明原因）

### 5. 发现一个问题，不要顺手修

这轮是 **审计模式**。先记问题，再统一汇报。

## 重点检查规则

### 1. 不要把“浏览器模式限制”误写成“页面没问题”

如果页面显示：
- “请通过 Tauri 启动”
- “浏览器模式不可用”

你要明确写：
- 这是降级提示
- 页面结构通过 / 功能未验证

### 2. 要区分三种情况

对每个问题写清楚它属于哪一种：

1. **页面 bug**：页面自身逻辑错
2. **数据绑定 bug**：后端有值，前端显示错
3. **环境限制**：浏览器模式天然做不到

### 3. 对“0/0”“—”“暂无数据”要特别敏感

你必须判断：
- 这是真空状态
- 还是假数据
- 还是绑定失败

不能直接把“暂无数据”当通过。

## 最终要求

这次你要把审计做成“老板打开就能知道到底哪里坏了”的级别。

不要写：
- “大致正常”
- “基本通过”
- “应该没问题”

- 哪一页
- 哪个组件
- 现象是什么
- 有没有真实点击过

只要你没真实点到，就写“未验证”，不要装作做过。
