# OpenClaw Bot 行为准则

> 类型: 架构/设计
> 最后更新: 2026-06-22
> 目标: 让 issue、PR、Discussions 和安全报告保持专业、友善、可复现。

## 一、我们的承诺

OpenClaw Bot 欢迎不同背景、经验水平和使用场景的贡献者。维护者承诺尽量维护一个开放、尊重、以问题解决为中心的协作环境。

## 二、鼓励的行为

- 用清晰、具体、可复现的方式描述问题。
- 尊重不同观点，先讨论事实和证据，再讨论方案取舍。
- 在日志、截图和复现材料中主动脱敏。
- 对新贡献者保持耐心，解释项目约定和验证方式。
- 发现安全问题时优先走私密漏洞报告入口，不在公开讨论中扩散细节。

## 三、不接受的行为

- 人身攻击、骚扰、歧视性语言或威胁。
- 在公开 issue、PR、评论或截图中发布 Key、token、Cookie、账号、二维码、验证码、交易凭证或隐私数据。
- 要求项目支持刷量、垃圾信息、绕过验证码、绕过登录保护、绕过付费墙、规避平台风控或违反服务条款的功能。
- 明知存在高风险，却要求移除人工确认、鉴权、白名单、脱敏或安全提示。
- 重复刷屏、恶意催促、偏离主题或蓄意消耗维护者时间。

## 四、执行方式

维护者可以根据严重程度采取以下动作：

1. 提醒对方修改措辞、补充复现或移除敏感信息。
2. 编辑、隐藏或删除包含敏感信息或不当内容的公开讨论。
3. 锁定讨论、关闭 issue / PR，或限制继续参与。
4. 对明显恶意或违法内容直接上报平台。

## 五、报告方式

如果你遇到行为准则问题：

- 普通协作问题：在相关 issue / PR 中保持简短、具体地说明。
- 涉及敏感信息或安全风险：使用仓库 Security 页面中的私密漏洞报告入口。
- 如果私密入口不可用：开一个不包含敏感细节的 issue，标题写 “Maintainer contact needed”。

## 六、适用范围

本行为准则适用于 OpenClaw Bot 仓库的 issue、PR、Discussions、安全报告、代码审查和围绕本项目的公开协作空间。
