## 关键发现（仍有参考价值）

### R01 基础设施

- Docker Redis `maxmemory`(256mb) 超过容器限制(128M)，已修为 100mb
- `packages/clawbot/.github/` 子目录 CI 不被 GitHub 触发，属冗余已删
- `.secrets.baseline` 需要与 pre-commit 配合使用
- `requirements.txt` 中 16 个 `>=` 无上限依赖需逐步加上限
- CI 缺少 ruff lint 和安全扫描步骤（建议迭代添加）

### R02 后端核心

- **Bug 修复**: `rpc.py` 缺 `import os`、WebSocket 多客户端消息丢失(共享 deque→独立 Queue)、WS 初始状态无异常保护
- **安全**: API Key 日志脱敏改用 SHA256 hash fingerprint；全部 SQLite 使用参数化查询+WAL 模式
- **技术债**: LLM 路由 JSON 配置与硬编码漂移(R2.25/R2.29)；无数据库备份策略(R2.36)；59 个静默异常(R2.44)；8 个孤儿 create_task(R2.45)

### R06/R07 macOS 桌面端

- 多处 Mock 硬编码假数据（资产分布、活动列表、订单簿、深度图、DevPanel）需替换为 API 调用
- 插件商店静默降级无用户提示(MOCK-06)
- 所有页面均有审计条目但多数为交互测试项，需实际运行验证

### R12 CI/CD

- **阻塞**: GitHub Actions Billing 问题导致所有 CI 运行失败，已增加本地替代方案 `make ci-local`
- 前端 npm 依赖未缓存、pytest 跑全量含已知失败、缺少路径过滤（均已修复）
- CI 失败无通知机制（低优先级，建议后续迭代添加 Telegram 通知）
